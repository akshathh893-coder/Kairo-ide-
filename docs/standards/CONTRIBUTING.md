# Contributing to KAIRO Cloud IDE

Thank you for helping build a better cloud development experience for iPad and beyond.

## Ground Rules

1. **Read before you write.** Start with the relevant specification or ADR.
2. **Architecture is not open for debate in a PR.** If you believe the design needs to change, open a discussion or draft an ADR first.
3. **Keep changes small and focused.**
4. **Tests and doctor must stay green.**
5. **No secrets. Ever.**

## Process

1. Find or create the specification under `docs/specifications/`.
2. Implement against it.
3. Open a pull request using the provided template.
4. Address CodeRabbit feedback without redesigning.
5. Merge only after CI is green and the specification acceptance criteria are met.

## Questions?

Open an issue. Prefer linking to the specific document you are asking about.
