# Implement Command

When the user asks you to implement a feature:

1. Identify the relevant specification under `docs/specifications/`.
2. If no specification exists, stop and say so. Do not invent one.
3. Read the SPEC.md (and any API.md / FLOW.md) carefully.
4. Implement exactly against the documented interfaces and acceptance criteria.
5. Do not rename public types, change method signatures, or introduce new top-level concepts.
6. If the specification is incomplete or contradictory, stop and explain the problem.
7. After implementation, update any relevant checklist in the specification or IMPLEMENTATION_QUEUE.md.
